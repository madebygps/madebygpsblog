The values for nice range from -20 (most likely to receive priority) to +19 (least likely to receive priory). </p><p>A high <code>nice</code> value translates to a low priority (when you <em>are</em> being nice to other users and processes), and a low <code>nice</code> value translates to a high priority (when you <em>are not</em> being so nice to other users and processes).</p><p>When a process is started, all standard processes are launched with a <code>nice</code> value of 0.</p><p>We can use the <code>nice</code> command to <em>set</em> the nice value when a process is launched and we can use <code>renice</code> to <em>adjust</em> the nice value of a running process.</p><p>When we start a process using the <code>nice</code> command but without providing any value, the default <code>nice</code> value is 10.</p><p>The <code>nice</code><strong><em> </em></strong>command requires that you increment the nice value, the <code>renice</code> command wants an absolute value for niceness. </p><h3 id="setting-the-priority-when-starting-a-process-with-nice">Setting the Priority When Starting a Process with <code>nice</code></h3><p>We can use the <code>nice</code> command to adjust the nice value for a program as we launch it. This allows us to increase or decrease the priority given to the process by the kernel, relative to the other processes.</p><p>On the left, I've execute <code>watch -n1 free</code> to show me details on the system’s memory usage.
