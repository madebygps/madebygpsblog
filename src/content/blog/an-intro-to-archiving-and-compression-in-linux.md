<p>Let's take a look at a few commands</p><pre><code class="language-bash">gzip
bzip2
tar</code></pre><h2 id="compressing-files">Compressing Files</h2><p>Data compression is the process of removing redundancy from data, done with a compression algorithm. These algorithms fall into two categories:</p><ul><li><strong>Lossless</strong>: Preserves all the data contained in the original. </li><li><strong>Lossy</strong>: Removes data as the compressions is performed to allow more compression to be applied. When the file is restored, it does not match the original version, it's a close approximation. JPEG and MP3 are examples of lossy compression.</li></ul><h3 id="the-gzip-command">The gzip command</h3><p>This command is used to compress one or more files. It replaces the original file with a compressed version.</p><p>The syntax of the command is <code>gzip file</code></p><figure class="kg-card kg-image-card"><img the cost of compression speed. </p><p>Use to compress:</p><pre><code class="language-bash">bzip2 sample.txt</code></pre><p>Use to uncompress:</p><pre><code class="language-bash">bunzip2 sample.txt.bz2</code></pre><p><code>bzip2</code> also has several supported options.</p><h2 id="archiving-files">Archiving Files</h2><p>Archiving is the process of gathering up many files and bundling them together into a single large file. </p><h3 id="the-tar-command">The tar command</h3><p>The <code>tar</code> program is a classic tool for archiving files. <!--kg-card-end: markdown--><p>I've created several folders and files, I can archive them with <code>tar cf playground.tar playground</code></p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-14.png" class="kg-image" alt loading="lazy" width="1770" height="688" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-14.png 600w, https://madebygps.azurewebsites.net/content/images/size/w1000/2021/03/image-14.png 1000w, https://madebygps.azurewebsites.net/content/images/size/w1600/2021/03/image-14.png 1600w, https://madebygps.azurewebsites.net/content/images/2021/03/image-14.png 1770w" sizes="(min-width: 720px) 720px"></figure><p>We can use the <code>v</code> option (verbose) for a more detailed listing</p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-15.png" class="kg-image" alt loading="lazy" width="1206" height="554" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-15.png 600w, https://madebygps.azurewebsites.net/content/images/size/w1000/2021/03/image-15.png 1000w, https://madebygps.azurewebsites.net/content/images/2021/03/image-15.png 1206w" sizes="(min-width: 720px) 720px"></figure><p>We can extract the <code>tar</code> in a new location. Keep in mind that unless we are operating as the superuser, files and directories extracted from archives take on the ownership of the user performing the restoration, rather than the original owner.</p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-16.png" class="kg-image" alt loading="lazy" width="913" height="332" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-16.png 600w, https://madebygps.azurewebsites.net/content/images/2021/03/image-16.png 913w" sizes="(min-width: 720px) 720px"></figure><p><code>tar</code> uses relative path names instead of absolute by default. This is useful because it allows us to extract archives to any location rather than having to extract them to their original locations. </p><p>You can limit what is extracted from the archive by adding the trailing <code>pathname</code> to the <code>tar</code> command and <code>tar</code> will only restore that specified file. You can also specify multiple path names. Wildcards are not normally supported, but the GUN version of <code>tar</code> does support them with the <code>--wildcards</code> option</p><pre><code class="language-bash">tar xf ../playground2.tar --wildcards 'home/azureuser/playground/dir-*/file-A'</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-17.png" class="kg-image" alt loading="lazy" width="1553" height="651" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-17.png 600w, https://madebygps.azurewebsites.net/content/images/size/w1000/2021/03/image-17.png 1000w, https://madebygps.azurewebsites.net/content/images/2021/03/image-17.png 1553w" sizes="(min-width: 720px) 720px"></figure><p>Modern versions of GNU <code>tar</code> support both <code>gzip</code> and <code>bzip2</code> compression directly with the use of the <code>z</code> and <code>j</code> options, respectively. </p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-18.png" class="kg-image" alt loading="lazy" width="1554" height="226" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-18.png 600w, https://madebygps.azurewebsites.net/content/images/size/w1000/2021/03/image-18.png 1000w, https://madebygps.azurewebsites.net/content/images/2021/03/image-18.png 1554w" sizes="(min-width: 720px) 720px"></figure><p>The <code>-T</code> is short for the <code>--files-from</code> option and it indicates to <code>tar</code> where to read its list of path names from, previously we told <code>tar</code> the directory or file to use but now we are using the <code>-</code> option and this let's <code>tar</code> know to use standard input or output as the files to extract or archive. The <code>-T</code> option is a little confusing to grasp at first. What this command is saying:</p><ol><li><code>find playground/ -name 'file-A</code> find all files in playground that match file-A </li><li><code>|</code> pipe what you found to <code>tar</code></li><li><code>tar czf playground</code> create a <code>gzip</code> compressed tar named playground.tgz </li><li><code>-T -</code> with the files from standard output, which are coming from the piped <code>find</code> command standard output.</li></ol>
