<p>Environment variables control how your working environment in Linux looks, acts, and feels. There are two types of variables:</p><ul><li><strong>Environment variables: </strong>These are process-wide variables built into the system and control the way it looks and acts, since they are process-wide, they are inherited by any child shells or processes. </li><li><strong>Shell variables: </strong>Are typically listed in lowercase and are only valid in the shell they are set in. </li></ul><h2 id="viewing-and-modifying-environment-variables">Viewing and Modifying Environment Variables</h2><p>Use the <code>env</code> command to view all you default environment variables:</p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-52.png" class="kg-image" alt loading="lazy" width="644" height="297" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-52.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-52.png 644w"></figure><p>Environment variables are always uppercase, as we can see <code>SHELL</code>, <code>WINDOWID</code> and <code>COLORTERM</code></p><p>To view all environment variables, including shell variables, local variables, and shell functions such as any user-defined variables and command aliases, we can use the <code>set</code> command. I'll use it with <code>more</code> so it's a bit more manageable to view:</p><p><code>set | more</code></p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-53.png" class="kg-image" alt loading="lazy" width="713" height="263" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-53.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-53.png 713w"></figure><h2 id="changing-variable-values">Changing Variable Values</h2><p>We can change a variable by providing the name, followed by an equals sign and the new value</p><pre><code class="language-bash">HISTSIZE=200</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-54.png" class="kg-image" alt loading="lazy" width="521" height="235"></figure><p>We've changed the value of <code>HISTSIZE</code> but only in our particular environment, if we wanted to make this change permanent we would use the <code>export</code> command. <code>export</code> will set the new value from your current environment to any new forked child processes, allowing new processes to inherit the exported variables. </p><pre><code class="language-bash">export HISTSIZE</code></pre><p><strong>NOTE:</strong> Before making any changes, you might want to save a copy of the current value <code>echo $VARIABLE&gt; ~/valueOfVARIABLE.txt</code> or you might even want to keep a copy of all current variables values <code>set&gt; ~/valueofALLon02252020.txt</code></p><p>Another variable you might want to modify is the <code>PS1</code> variable. It has a set of placeholders for information you want to display in your shell prompt:</p><ul><li>\u The name of the current user</li><li>\h The hostname</li><li>\w The base name of the current working directory </li></ul><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-55.png" class="kg-image" alt loading="lazy" width="735" height="217" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-55.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-55.png 735w" sizes="(min-width: 720px) 720px"></figure><h2 id="the-path-variable">The PATH variable</h2><p>The <code>PATH</code> variable controls where on your system your shell will look for commands. If the bash shell doesn't find the command in one of the directories in your <code>PATH</code> variables, it will return the error <code>command not found</code></p><p>Let's see which directories are stored in your <code>PATH</code> variable </p><pre><code class="language-bash">echo $PATH</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-56.png" class="kg-image" alt loading="lazy" width="736" height="216" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-56.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-56.png 736w" sizes="(min-width: 720px) 720px"></figure><p>There are the directories where my terminal will search for any command. Each directory is separated by a colon (:)</p><p>Let's say I have some commands in the <code>/home/azureuser/gpstools</code> directory that I would add to my <code>PATH</code> We can add a directory to the <code>PATH</code> variable by entering</p><pre><code class="language-bash">PATH=$PATH:/directory/to/add</code></pre><p>This will assign the original <code>PATH</code> plus the <code>/directory/to/add/</code> </p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-57.png" class="kg-image" alt loading="lazy" width="1502" height="244" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-57.png 600w, https://madebygps.azurewebsites.net/content/images/size/w1000/2021/02/image-57.png 1000w, https://madebygps.azurewebsites.net/content/images/2021/02/image-57.png 1502w" sizes="(min-width: 720px) 720px"></figure><p>DO NOT DO <code>PATH=/directory/to/add</code> that will replace everything and add the new directory, you want to append, not replace.</p><h2 id="creating-a-user-defined-variable">Creating a User-Defined Variable</h2><p>We can create our own variables by following this syntax</p><pre><code class="language-bash">MYVARIABLE="VALUE"</code></pre><p>If you need it to persist, don't forget to export it. <code>export MYVARIABLE</code> and if you want to delete any variable, use the <code>unset</code> command, just double check before deleting anything. <code>unset MYVARIABLE</code></p>
