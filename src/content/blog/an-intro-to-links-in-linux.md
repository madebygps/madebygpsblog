<p>In Unix-like systems, it is possible to have a file reference by multiple names. We use links for this, there are two types.</p><h2 id="hard-links">Hard Links</h2><p><code>Hard links</code> also allow to point to files, but in a different way than symbolic links. They are the original Unix way of creating links. By default every file has a single hard link that gives the file its name, when we create a <code>hard link</code>, we are creating an additional directory entry for a file. There are two limitations:</p><ul><li>A <code>hard link</code> cannot reference a file outside of its own file system. The file it is referencing has to be on the same disk partition. </li><li>A <code>hard link</code> cannot reference a directory. </li></ul><p>A <code>hard link</code> is indistinguishable from the file itself. I've listed a directory that contains a <code>hard link</code> and a <code>symbolic</code> link below and we can see that the <code>hard link</code> does not have a special indication that it's a hard link, the <code>symbolic link</code> does (the <code>l</code> in the permissions and the <code>-&gt;</code> showing what it links to)</p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-19.png" class="kg-image" alt loading="lazy" width="1308" height="235" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-19.png 600w, https://madebygps.azurewebsites.net/content/images/size/w1000/2021/03/image-19.png 1000w, https://madebygps.azurewebsites.net/content/images/2021/03/image-19.png 1308w" sizes="(min-width: 720px) 720px"></figure><p>When a <code>hard link</code> is deleted, the link is removed, but the content of the file itself continue to exist until all links to the file are deleted. You may encounter them from time to time, but modern practice prefers symbolic links. </p><p>We can create a hard link by using the following command</p><pre><code class="language-bash">ln file link</code></pre><h2 id="symbolic-links">Symbolic Links</h2><p>Symbolic links are a special type of file. They are also called <code>soft links</code> or <code>symlinks</code> They are used to point to any file or directory on any computer, similar to a shortcut in Windows and an alias in Mac OS.</p><p>We can create a symbolic link with</p><pre><code class="language-bash">ln -s item link</code></pre><p>and item can be either a file or directory.</p><p>Symbolic links overcome the limitations of hard links because they work by creating a special type of file that contains a text pointer to the referenced file or directory.</p><p>If you write something to the symbolic link, the referenced file is written to, when you delete the symbolic link, only the link is deleted and not the file itself. If the file is deleted, the link will continue to exist but will point to nothing and is said to be broken. </p>
