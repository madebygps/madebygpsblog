<p>I/O redirection allows us to redirect input and output of commands to and from files, as well as connect multiple commands together into pipelines.</p><h2 id="standard-input-output-and-error">Standard Input, Output, and Error</h2><p>Output consists of two types</p><ul><li>The program's results. Known as Standard Output or <code>stdout</code></li><li>Status and error messages. Known as Standard Error or <code>stderr</code></li></ul><p>In addition, some programs take input from standard input <code>stdin</code> which by default is attached to the keyboard. </p><p>By default, both <code>stdout</code> and <code>stderr</code> are linked to the screen and not saved into a file. Redirection allows us to change where output goes and where input comes from. </p><h2 id="redirecting-standard-output">Redirecting Standard Output</h2><p>To redirect <code>stdout</code> to another file instead of the screen, we use the <code>&gt;</code> redirection operator followed by the name of the file.</p><p>For example, we could tell the shell to send the output of the <code>ls</code> command to the file <code>ls-output.txt</code> instead of the screen</p><pre><code class="language-bash">ls -l /usr/bin &gt; ls-output.txt</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-8.png" class="kg-image" alt loading="lazy" width="823" height="464" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-8.png 600w, https://madebygps.azurewebsites.net/content/images/2021/03/image-8.png 823w" sizes="(min-width: 720px) 720px"></figure><p>We can see that the ls output was not sent to the screen, but to the <code>ls-output.txt</code> file.</p><p>Keep in mind that using the redirection operator will overwrite the destination file. To append, we use the <code>&gt;&gt;</code> redirection operator. </p><h2 id="redirecting-standard-error">Redirecting Standard Error</h2><p>To redirect <code>stderr</code> we must refer to its file descriptor. The shell references <code>stdout</code> <code>stdin</code> and <code>stderr</code> internal as file descriptors 0, 1, and 2, respectively. We can redirect <code>stderr</code> with this notation:</p><pre><code class="language-bash">ls -l /bin/usr 2&gt; ls-error.txt</code></pre><h2 id="redirecting-standard-output-and-standard-error-to-one-file">Redirecting Standard Output and Standard Error to One File</h2><p>There are two ways to accomplish these, first let's use the traditional method, which works with old versions of the shell:</p><pre><code class="language-bash">ls -l /bin/usr &gt; ls-output.txt 2&gt;&amp;1</code></pre><p>First we are redirecting <code>stdout</code> to the file ls-output.txt and then we redirect file descriptor 2 <code>stderr</code> to file descriptor 1 <code>stdout</code> using the <code>2&gt;&amp;1</code> notation. </p><p>Keep in mind that the order of the redirections matter, the redirection of <code>stderr</code> must always occur after redirecting <code>stdout</code>.</p><p>Recent versions of bash provide a second, more streamlined method for performing this combined redirection</p><pre><code class="language-bash">ls -l /bin/usr &amp;&gt; ls-output.txt</code></pre><p>You can still append using the <code>&gt;&gt;</code></p><h2 id="disposing-of-unwanted-output">Disposing of Unwanted Output</h2><p>The system provides a way to redirect output to a special file called <code>/dev/null</code> which is often referred to as a <code>bit bucket</code>. It accepts input and does nothing with it.</p><pre><code class="language-bash">ls -l /bin/usr 2&gt; /dev/null</code></pre><h2 id="redirecting-standard-input">Redirecting Standard Input</h2><p>Using the <code>&lt;</code> redirection operator, we can change the source of <code>stdin</code> from the keyboard to a file.</p><pre><code class="language-bash">cat &lt; sample.txt</code></pre><h2 id="pipelines">Pipelines</h2><p>Using the pipe operator <code>|</code>, the <code>stout</code> of one command can be piped into the <code>stdin</code> of another. <code>less</code> is an example of this</p><pre><code class="language-bash">ls -l /usr/bin | less</code></pre><p>it is possible to put several commands together into a pipeline. The commands used this way are referred to as filters. Filters take input, change it somehow and then output it.</p><h3 id="the-tee-command">The <code>tee</code> command</h3><p>The <code>tee</code> command reads <code>stdin</code> and copies it to both <code>stdout</code> and to one or more files. </p><pre><code class="language-bash">ls /usr/bin | tee ls.txt | grep zip</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-9.png" class="kg-image" alt loading="lazy" width="930" height="625" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-9.png 600w, https://madebygps.azurewebsites.net/content/images/2021/03/image-9.png 930w" sizes="(min-width: 720px) 720px"></figure>
