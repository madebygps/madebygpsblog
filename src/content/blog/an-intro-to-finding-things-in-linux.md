<p>Let's take a look at these commands:</p><pre><code class="language-shell">locate
whereis
which
find</code></pre><h2 id="the-locate-command">The <code>locate</code><em> </em>command</h2><p>This command will go through your entire filesystem and locate every occurrence of that keyword, so you can image that the results can be overwhelming.</p><pre><code class="language-shell">locate keyword</code></pre><p><code>locate</code><em> </em>uses a database that is usually updated once a day, so if you're searching for something that was created recently, it might not return in your search. You can use the </p><pre><code class="language-shell">updatedb</code></pre><p>command to manually update the <code>locate</code><em> </em>command's database.</p><pre><code class="language-shell">locate aircrack-ng</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-1.png" class="kg-image" alt loading="lazy" width="908" height="382" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-1.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-1.png 908w" sizes="(min-width: 720px) 720px"></figure><h2 id="the-whereis-command">The <code>whereis</code><em> </em>command</h2><p>In Linux, executable files are called binaries, if you want to locate a binary, <code>whereis</code><em> </em>is more efficient than <code>locate</code>.</p><pre><code class="language-shell">whereis binary</code></pre><p>This command will return the binaries location, its source and the man page if available</p><pre><code class="language-shell">whereis aircrack-ng</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-2.png" class="kg-image" alt loading="lazy" width="960" height="108" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-2.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-2.png 960w" sizes="(min-width: 720px) 720px"></figure><h2 id="the-which-command">The <code>which</code><em> </em>command</h2><p>The <code>PATH</code><em> </em>variable in Linux holds the directories in which the operating system looks for the commands you execute in the command line. </p><pre><code class="language-shell">which binary</code></pre><p>The which command locates an a binary in your <code>PATH</code>. If it doesn’t find the binary in the current <code>PATH</code>, it returns nothing. </p><pre><code class="language-shell">which aircrack-ng</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-3.png" class="kg-image" alt loading="lazy" width="959" height="202" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-3.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-3.png 959w" sizes="(min-width: 720px) 720px"></figure><p>These directories typically include <code>/usr/bin</code> but may include <code><em>/</em>usr/sbin</code> and a few others. </p><h2 id="the-find-command">The <code>find</code><em> </em>command</h2><p>The most powerful searching command is the <code>find</code><em> </em>command. You can use it to search in any designated directory and use a variety of parameters.</p><p>The basic syntax is:</p><pre><code class="language-shell">find directory options expression</code></pre><p>Let's say I have a file named <code>test.txt</code> and I need to find it but am not sure what exact directory it's in. I can execute the following to search starting from the top of the file system / </p><pre><code class="language-shell">find / -type f -name test.txt</code></pre><ul><li><code>/</code> means from the top of the file system</li><li><code>-type</code> is what you are looking for, <code>f</code> means file, <code>b</code> means block special device file, <code>c</code> character special device file, <code>d</code> directory, <code>l</code> symbolic link.</li><li><code>-name</code> is the name you are looking for, results will match exactly. </li></ul><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-4.png" class="kg-image" alt loading="lazy" width="958" height="161" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-4.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-4.png 958w" sizes="(min-width: 720px) 720px"></figure><p>A search that looks in every directory, starting from the top, can take a while. We can speed things up by providing a directory, let's say I know this file is in the home directory.</p><pre><code class="language-shell">time find /home -type f -name test.txt</code></pre><p>I used the <code>time</code> command here so we can see how long each command took.</p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-5.png" class="kg-image" alt loading="lazy" width="961" height="355" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-5.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-5.png 961w" sizes="(min-width: 720px) 720px"></figure><p>The <code>find</code> command only displays exact name matches. If <code>file.txt</code> had a different extension, it would not have been returned. I've created another file <code>test.conf</code><em> </em>and now if I search with <code>find</code> only using <code>test.txt</code> as the name, I no longer get the <code>test.conf</code> file returned.</p><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-7.png" class="kg-image" alt loading="lazy" width="959" height="150" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-7.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-7.png 959w" sizes="(min-width: 720px) 720px"></figure><p>We can work around this limitation by using <code>wildcards</code><em>. </em>They let us match multiple characters and come in a few different forms: </p><p>Let's say we have a directory with files cat, hat, what, and bat</p><ul><li><code>*</code> matches multiple characters <code>*at</code> would match: cat, hat, what, and bat.</li><li><code>?</code> matches a single character <code>?at</code> would match cat, hat, bat but not what. </li><li><code>[]</code> matches character that appear inside the square brackets <code>[c,b]</code> would match cat and bat</li></ul><pre><code class="language-shell">find /home -type f -name test.*</code></pre><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/02/image-8.png" class="kg-image" alt loading="lazy" width="957" height="146" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/02/image-8.png 600w, https://madebygps.azurewebsites.net/content/images/2021/02/image-8.png 957w" sizes="(min-width: 720px) 720px"></figure><p><code>find</code> supports plenty of tests and even operators, let's say we wanted to find all the files with permissions that are not 0600 and the directories that are not 0700.</p><pre><code class="language-bash">find ~ \( -type f -not -perm 0600 \) -or \( -type d -not perm 0700 \)</code></pre><p>this command says: find all files where permissions are not 0600 or all directories where permissions are not 0700.</p><ul><li><code>find ~</code> look in the <code>~</code> directory (home).</li><li><code>\( -type f -not -perm 0600)</code> The slash is escaping the parenthesis, we use parenthesis here to group tests and operators together to form a larger expression. By default, <code>find</code> evaluates from left to right. The <code>-not</code> tells us this test is a match if the result is false. <code>-not</code> can be abbreviated with an <code>!</code> so this part could be <code>\( -type f ! -perm 0600)</code> as well</li><li><code>-or</code> This is telling us that if either test is true, it's a match. Can be abbreviated to <code>-o</code> </li><li><code>\( -type d -not perm 0700 \)</code>another test, very similar to the first one, except the type here is directory.</li></ul><figure class="kg-card kg-image-card"><img src="https://madebygps.azurewebsites.net/content/images/2021/03/image-7.png" class="kg-image" alt loading="lazy" width="861" height="113" srcset="https://madebygps.azurewebsites.net/content/images/size/w600/2021/03/image-7.png 600w, https://madebygps.azurewebsites.net/content/images/2021/03/image-7.png 861w" sizes="(min-width: 720px) 720px"></figure><p><code>find</code> is a powerful command with many tests, make sure to look into it more.</p><p>and that's it for this intro to finding stuff in Linux :)</p>
